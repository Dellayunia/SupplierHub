Segmen A: Validasi Modul Autentikasi & Sesi Otentikasi
B01 **Login Valid:** Input Email dan Sandi sesuai dengan data rekam *database LBS*. Sistem menerima kredibilitas pengguna dan menavigasikan aksi layar (*Route*) ke halaman *Beranda (Home)*. Aplikasi sukses berpindah dari `AuthStack` masuk ke menu antarmuka utama sistem. Valid
B02 **Autentikasi Sandi Salah:** Input Email benar, namun komponen input Sandi disalahkan acak. Sistem memutus upaya perpindahan *Route* dan melempar notifikasi (*Error Alert*) kepada pengguna. *State* ditolak masuk; muncul notifikasi pemberitahuan akses silang ditolak. Valid
B03 **Blokade Register Kosong:** Menekan instrumen aksi *"Daftar"* tanpa mengisi identitas apa pun. Atribut validasi (*form guard*) wajib menahan *request payload* agar tidak dikirim ke *backend*. Aplikasi diam di tempat dan baris formulir memberikan garis peringatan warna merah. Valid
B04 **Penghancuran Sesi (Logout):** Menekan tombol keluar (Log Out) pada antarmuka *Settings/Profile*. Sistem operasi lokal harus menghapus memori *token / cache* dan melempar akses *Route* kembali ke layar masukan. *Token* pengguna musnah dari aplikasi lokal, navigasi segera terisolasi ke halaman *Login*. Valid
Segmen B: Validasi Komputasi Algoritmis & Geospasial
B05 **Bypass Radius Koordinat:** *Backend* diuji kelayakan permintaan dengan mendesak parameter asimtot di atas radius statik 5.000 meter. Sistem perantara Node.js wajib menolak instruksi iterasi API jika batas melebihi absolut 5 Km. API memutus iterasi rute *Places* yang melanggar batas maksimum konfigurasi (5.000m). Valid
B06 **Hitung Geometris Jarak:** Sinkronisasi visual koordinat garis lengkung bumi via Formula Haversine dari titik asal ke kandidat POI. Angka metrik jarak di dalam kartu UI wajib selaras membandingkan komputasi titik `Lat/Lng` dengan logika visual peta bumi nyata. Aplikasi mencetak hasil parameter kilometer yang selaras, terhindar komputasi salah Euclidean linier datar. Valid
B07 **Kalkulasi Silang Matriks SAW:** Kasus ekstrem sistem dihadapkan kandidat *Mitra A* (Sangat Dekat) berlawanan dengan *Mitra B* (*Rating* Publik Sangat Tinggi). *Logic* sistem wajib memangkas dan memenangkan kriteria letak (Jarak) akibat preferensi faktor bobot kemutlakan $W$ (0.6). Skor akhir ($V_i$) sukses meletakkan peringkat #1 pada sasaran bernilai jarak kurir minimum. Valid
B08 **Navigasi Detail Entitas:** Pengguna memijat komponen *Card Ranking* untuk membaca riwayat/ulasan detail spesifik milik *Supplier*. Komponen navigasi React Native mentransisikan referensi *params ID* kandidat secara mutlak menuju halaman baru yang memuat metadatenya. Halaman *Detail Partner* terbaca sinkron sesuai *ID* pilihan yang diklik tanpa pertukaran identitas palsu. Valid
Segmen C: Evaluasi Sintesis Gemini LLM & Interupsi Skrip
B09 **Injeksi Parameter Otomatis (Prompting):** Implementasi penekanan tindakan pembuat (*Generate*) Proposal Bisnis menggunakan skrip AI Generatif LLM. Variabel letak subjek UMKM (identitas profil, nama target rekanan) diekstraksi mulus ke dalam JSON parameter LLM Google Gemini. *Prompt* paramater tereksekusi, *Output* teks proposal tercipta menggunakan gaya Bahasa Bisnis Formal sesuai kustomisasi lokal awal. Valid
B10 **Anomali Respons Interupsi Waktu (Latency):** Kasus dimana panggilan API LLM mengekstrak komputasi tinggi memakan *delay* ekstrem (di atas 3 detik). Aplikasi tidak boleh lumpuh statis (*Force Close/ANR*). Modul UI wajiib memicu *animation loading* seketika untuk meredam persepsi kebosanan. Skrip asinkrosnus sukses menahan *State* dengan *Spinner Indicator* hingga respons API ditransmisikan (~4 Detik latensi pemulihan). Valid
B11 **Penahanan Galat Kuota API LLM:** Jika transmisi token akses Gemini tidak mampu tersambung karena kelebihan permintaan limit (*Rate Limit*). Perintah instruksi akan terhenti terkunci dan UI merilis *Error Handing System* ramah pengguna (*"Network Error, Try Again"*). Rejeksi *Token/Limits API* ditangkap oleh siklus *Catch/Try Block* aman pada React Native. Valid
Segmen D: Fungsionalitas Umum (CRUD) & Menu Status
B12 **Manajemen Tambah Katalog:** Pemilik usaha mencoba menambahkan jenis SKU list barang industri baru via halaman Pengaturan (*Settings*). Integrasi *form* mencetak objek rekam lokal ke instumen Firebase/Basis Data agar produk terlihat subjek audiens pada aplikasi. Entitas Katalog baru di Beranda tersambung masuk dan tertambahkan otomatis di hitungan layar muka. Valid
B13 **Visualisasi Maps Engine:** Beranjak melihat mode "Peta Mitra" (Bottom Tab Maps Layout). Pustaka perpustakaan grafis merender kanvas Google Maps *Embed SDK* mendistribusikan *Pin Markers* lokasi *Supplier* pada radius sekitar asimtotik. Peta spasial termuat lancar tanpa grafik cacat memfokuskan lokasi pusat *Handphone* GPS terkalibrasi ke dalam sentris pandang awal pemakai. Valid
B14 **Pemantauan Interupsi Bottom-Tab:** Pengguna memintas pergantian menu navigasi antarmuka tabung bawah berulang kali (Home <-> Catalog <-> Notifications <-> Maps). *Router App Stack* mempertahankan statis *lifecycle event* menghindari ter-relokasinya antarmuka (cegah *Memory Leakage* gawai memori rendah). Perpindahan aplikasi asinkron berjalan cepat (instan), aman dari keruntuhan antarmuka (*render collapse*). Valid
